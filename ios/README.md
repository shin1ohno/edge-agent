# WeaveIos

Native iOS/iPad app: runs as a portable edge (CoreBluetooth → Nuimo → Rust core) and provides weave-web–equivalent operations on-device.

## One-time Mac setup

```bash
# 1. Rust iOS targets
rustup target add aarch64-apple-ios aarch64-apple-ios-sim x86_64-apple-ios

# 2. XcodeGen (generates WeaveIos.xcodeproj from project.yml)
brew install xcodegen

# 3. Xcode command-line tools + first-launch init
sudo xcode-select -s /Applications/Xcode.app/Contents/Developer
sudo xcodebuild -runFirstLaunch
```

## Build loop

```bash
cd ios

# 1. Build WeaveIosCore.xcframework + Swift bindings from Rust.
./build-xcframework.sh

# 2. (Re)generate WeaveIos.xcodeproj from project.yml.
xcodegen

# 3. Open Xcode, pick your Personal Team in Signing & Capabilities,
#    then Cmd+R to run on a connected iPhone/iPad.
open WeaveIos.xcodeproj
```

Step 1 only needs re-running when Rust sources under `crates/weave-ios-core/` or `crates/nuimo-protocol/` change. Step 2 only when `project.yml` or Swift source layout changes. Xcode picks up Swift source edits automatically.

## Personal Team re-sign every 7 days

With a free Apple ID, the Personal Team provisioning profile is valid for 7 days. On day 7:

1. Plug the device into the Mac that signed it originally.
2. `xcodegen && open WeaveIos.xcodeproj`
3. Cmd+R — Xcode re-signs and reinstalls in ~30 seconds.

The app keeps its data (UserDefaults, known Nuimo UUIDs) across re-installs.

## Directory layout

```
ios/
  project.yml                    # XcodeGen spec (tracked)
  build-xcframework.sh            # Rust → xcframework (tracked)
  deploy.sh                       # CLI build+install+launch on physical device (tracked)
  Frameworks/                     # generated xcframework (.gitignored)
  WeaveIos/
    WeaveIosApp.swift             # @main SwiftUI App
    Info.plist                    # generated by xcodegen from project.yml
    Core/
      BleBridge.swift             # CBCentralManager + delegate
      NuimoDevice.swift           # per-peripheral connection + event parse
      SettingsStore.swift         # UserDefaults-backed config
      UiClientHost.swift          # @Observable wrapper over UniFFI UiClient
      WeaveTypes.swift            # Codable mirrors of weave-contracts UiFrame / UiSnapshot
    Views/
      RootView.swift              # TabView root
      ConnectionsView.swift       # Server status + Edges / Zones / Lights (from /ws/ui)
      MappingsListView.swift      # Mappings list (read-only, Phase 5 MVP)
      HomeView.swift              # Nuimo BLE scan / connect / event feed / LED test
      GlyphListView.swift         # Glyph catalog entry points
      GlyphEditorView.swift       # 9×9 tap-to-toggle editor
      SettingsView.swift          # server URL + paired list
    Bundle/
      WeaveIosCore.swift          # generated Swift bindings (.gitignored)
```

`WeaveIos.xcodeproj/` is regenerated by `xcodegen` and gitignored.

## Tabs & what they need

| Tab          | What it needs to populate                               |
|--------------|---------------------------------------------------------|
| Connections  | `serverURL` set in Settings → WS `/ws/ui` reachable     |
| Mappings     | same as Connections (reads `UiSnapshot.mappings`)       |
| Nuimo        | physical device (simulator has no Bluetooth stack)       |
| Glyphs       | offline-capable; save-to-server lands with full RoutesEditor |
| Settings     | always available                                        |

## Simulator loop (UI / WS / REST verification — no BLE)

```bash
cd ios
./build-xcframework.sh     # only if Rust sources changed
xcodegen                    # if project.yml changed

# Build + install + launch on a booted simulator:
UDID=$(xcrun simctl list devices booted | awk -F'[()]' '/Booted/{print $2; exit}')
xcodebuild -project WeaveIos.xcodeproj -scheme WeaveIos -configuration Debug \
    -destination "id=$UDID" build
APP=$(xcodebuild -project WeaveIos.xcodeproj -scheme WeaveIos -configuration Debug \
    -destination "id=$UDID" -showBuildSettings 2>/dev/null \
    | awk -F= '/ CONFIGURATION_BUILD_DIR /{gsub(/^[ \t]+|[ \t]+$/,"",$2); print $2"/WeaveIos.app"}' | head -n1)
xcrun simctl install "$UDID" "$APP"
xcrun simctl launch "$UDID" com.shin1ohno.weave.WeaveIos
```

`ios/deploy.sh` covers the physical-device equivalent (Personal Team signing + `xcrun devicectl`).
